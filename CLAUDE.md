## 프로젝트 한 줄 요약

Windows "보내기(Send To)" 메뉴에서 동작하는, ZIP 전용 초경량 압축/압축 해제 도구.
속도 우선(SIMD 가속) → 그 안에서 압축률 향상. 비밀번호/암호화 기능 없음.

---

## 절대 원칙 (위반 시 작업 중단하고 보고)

1. **panic 금지**: `compress`, `extract` 등 코어 함수 내부에서 `unwrap()` / `expect()` / `panic!()` 사용 금지. 모든 실패 가능 지점은 `?`로 `AppError`에 흡수.
2. **테스트를 임의로 약화하지 않음**: 구현 중 테스트가 실패하면 **구현을 수정**한다. 테스트의 assert를 완화하거나, 테스트를 삭제/skip하거나, 테스트를 구현에 맞춰 바꾸는 방식으로 "통과"시키지 않는다. 테스트 자체가 잘못되었다고 판단되면 그 이유를 먼저 설명하고 합의 후 수정한다.
3. **ZIP 외 포맷 미지원**: 7z, RAR 등 다른 포맷의 읽기/쓰기 코드를 추가하지 않는다.
4. **암호화/비밀번호 기능 추가하지 않음**: 스코프 외로 확정된 사항. 관련 요청이 있어도 SPEC.md 변경 없이는 구현하지 않는다.
5. **설정파일을 만들지 않음**: 압축 레벨, STORE 확장자 목록 등은 `config.rs`의 상수로만 관리. 런타임에 파일을 읽어 설정을 로드하는 코드를 추가하지 않는다.
6. **콘솔 출력에 의존하지 않음**: `println!`/`eprintln!`로 사용자에게 정보를 전달하지 않는다. 사용자 대상 메시지는 `MessageBoxW` 경로로만.

## 기술 스택 / 의존성

- 언어: Rust (edition 2021+), 빌드: `cargo build --release` (단일 정적 exe, 런타임 의존성 없음)
- 주요 크레이트: `zip`, `libdeflater`, `walkdir`, `thiserror`, `windows`
- 새 의존성 추가 시: "가벼움" 원칙에 부합하는지(바이너리 크기, 컴파일 시간) 먼저 언급하고 진행

---

## 코드 컨벤션

- 사용자에게 보여지는 모든 문자열(에러 메시지, 알림 등)은 **한글**
- 코드 주석/식별자는 영문 관례 유지 (Rust 표준에 맞춤)
- 에러 타입은 `AppError`(thiserror) 하나로 통일, variant는 SPEC.md 7.1 기준
- 코어 로직(`compress`/`extract`, 경로 처리)과 셸/UI 레이어(MessageBoxW, argv 파싱, Send To 설치)는 모듈로 분리 — 코어는 향후 COM 셸 확장(방법 B) 전환 시 재사용 가능해야 함

---

## 테스트 정책

- **코어 로직은 TDD**: 테스트를 먼저 작성 → 실패 확인 → 구현 → 통과 확인 순서로 진행
- 핵심 테스트는 **라운드트립**(`compress` 후 `extract`하여 원본과 동일한 디렉토리 트리가 복원되는지) — SPEC.md 13.1 참조
- 셸/UI 레이어(Send To 설치, MessageBoxW 표시)는 자동 테스트 대상이 아니며, SPEC.md 13.2의 체크리스트로 수동 확인
- 마일스톤 완료 시마다 `cargo test`와 `cargo clippy`를 실행하고 결과를 보고할 것 (경고 0건 목표)

## 작업 진행 방식

- PLAN.md의 마일스톤 단위로 진행. 한 번에 여러 마일스톤을 동시에 구현하지 않음
- 각 마일스톤 종료 시: (1) 테스트 결과, (2) 자가 검토(절대 원칙 위반 여부, `unwrap`/`expect` 잔존 여부) 보고
- SPEC.md와 실제 구현이 달라지는 부분이 생기면, 임의로 진행하지 말고 먼저 알리고 SPEC.md 갱신 여부를 확인
- SPEC.md 14절("확인 필요 사항")의 미결 항목은 구현 전에 먼저 질문할 것
